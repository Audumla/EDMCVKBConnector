## System & Session Events

### `FileHeader`
**Frequency:** startup only | **Data:** Game version, build, language
- `timestamp` - Event timestamp
- `version` - Journal format version
- `language` - Game language setting
- `gameversion` - Game version
- `build` - Build number

### `Continued`
**Frequency:** per session | **Data:** Continuation of journal
- Indicates this is a continuation of an existing journal file (player didn't restart the game)

### `LoadGame`
**Frequency:** on login | **Data:** Complete commander & ship state
- `timestamp`
- `Commander` - Commander name
- `Horizons` - Boolean, has Horizons DLC
- `Odyssey` - Boolean, has Odyssey DLC
- `GameMode` - Open/Solo/Group
- `Group` - Private group name (if applicable)
- `Credits` - Starting credits for session
- `Loan` - Outstanding loan amount (if any)

### `ClearSavedGame`
**Frequency:** rare | **Data:** Game reset/new character
- `timestamp`
- `Commander` - Commander name being cleared

### `NewCommander`
**Frequency:** rare | **Data:** New character created
- `timestamp`
- `Commander` - New commander name
- `FID` - Frontier ID assigned
- `Package` - Edition (Deluxe, Standard, etc.)

### `Commander`
**Frequency:** on login | **Data:** Commander details
- `timestamp`
- `Name` - Commander name
- `FID` - Frontier ID
- `Rank` - Numeric (deprecated, use specific rank types)

### `Progress`
**Frequency:** on login | **Data:** Commander rank progress
- `timestamp`
- `Combat` - Combat rank % progress (0-100)
- `Trade` - Trade rank % progress
- `Explore` - Exploration rank % progress
- `Empire` - Empire rank % progress
- `Federation` - Federation rank % progress
- `CQC` - CQC rank % progress

### `Rank`
**Frequency:** on login | **Data:** All commander ranks
- `timestamp`
- `Combat` - Combat rank level (0-8)
- `Trade` - Trade rank level (0-8)
- `Explore` - Exploration rank level (0-8)
- `Empire` - Empire rank level (0-14)
- `Federation` - Federation rank level (0-14)
- `CQC` - CQC rank level (0-8)

### `Materials`
**Frequency:** on login | **Data:** Engineering material inventory
- `timestamp`
- Raw materials list (category name -> [[name, count], ...])
- Manufactured materials list
- Encoded materials list

### `Cargo`
**Frequency:** on login | **Data:** Cargo manifest
- `timestamp`
- Array of cargo items with quantities

### `Missions`
**Frequency:** on login | **Data:** Active missions
- `timestamp`
- Array of active mission details

### `Passengers`
**Frequency:** on login | **Data:** Passenger cabins
- `timestamp`
- Array of passenger details

### `Statistics`
**Frequency:** on login | **Data:** Career statistics
- `timestamp`
- Multiple statistics categories (Bank Account, Combat, Trading, Mining, Exploration, etc.)

## Travel & Navigation Events

### `Location`
**Frequency:** on startup if already flying | **Data:** Player location
- `timestamp`
- `StarSystem` - System name
- `SystemAddress` - Unique system ID64
- `StarPos` - [X, Y, Z] coordinates
- `Body` - Current body/station name
- `BodyID` - Body ID
- `BodyType` - Type (Star, Planet, Station, etc.)
- `StationType` - Station type (if in station)
- `StationName` - Station name (if docked)
- `StationFaction` - Controlling faction
- `DistFromStarLS` - Distance from main star in light-seconds
- `Docked` - Boolean, is docked

### `FSDJump`
**Frequency:** per hyperspace jump | **Data:** Jump completed
- `timestamp`
- `StarSystem` - Destination system name
- `SystemAddress` - Destination ID64
- `StarPos` - [X, Y, Z] coordinates
- `Economy` - Economy type
- `EconomySecond` - Secondary economy
- `Government` - Government type
- `Allegiance` - Superpower allegiance
- `Faction` - Controlling faction
- `FactionState` - Faction state
- `Security` - Security level

### `CarrierJump`
**Frequency:** per fleet carrier jump | **Data:** Fleet carrier jump
- `timestamp`
- `StarSystem` - Destination system
- `SystemAddress` - Destination ID64
- `StarPos` - Coordinates
- `Docked` - Boolean

### `StartJump`
**Frequency:** when FSD initiates | **Data:** Jump startup
- `timestamp`
- `JumpType` - Hyperspace or Supercruise
- `StarSystem` - Target system (may be empty for supercruise)
- `SystemAddress` - Target system ID (if hyperspace)

### `SupercruiseEntry`
**Frequency:** entering supercruise | **Data:** Supercruise engaged
- `timestamp`

### `SupercruiseExit`
**Frequency:** exiting supercruise | **Data:** Supercruise disengaged
- `timestamp`
- `StarSystem` - Current system
- `SystemAddress`
- `Body` - Body exited to near
- `BodyType`

### `FSDTarget`
**Frequency:** FSD target selected | **Data:** Jump target set
- `timestamp`
- `Name` - Target system name
- `SystemAddress` - Target system ID64

### `NavRoute`
**Frequency:** route plotted | **Data:** Full multi-jump route
- `timestamp`
- `Route` - Array of waypoints [{ StarSystem, SystemAddress, StarPos, StarClass }, ...]

### `NavRouteClear`
**Frequency:** route cancelled | **Data:** Route cleared
- `timestamp`

### `ApproachBody`
**Frequency:** entering body orbit | **Data:** Body approach
- `timestamp`
- `StarSystem`
- `SystemAddress`
- `Body` - Body name
- `BodyID` - Body ID

### `LeaveBody`
**Frequency:** leaving body orbit | **Data:** Body departure
- `timestamp`
- `StarSystem`
- `SystemAddress`
- `Body` - Body name

### `SupercruiseDestinationDrop`
**Frequency:** dropped at destination | **Data:** Dropped from supercruise
- `timestamp`

### `USSDrop`
**Frequency:** USS signal found | **Data:** Unidentified signal scanned
- `timestamp`
- `USSType` - Type of USS
- `USSThreat` - Threat level (0-4)
- `Interdict` - Boolean

## Synthetic Events

These are generated by EDMC itself when the game is running but no journal events are received.

### `StartUp`
**Frequency:** startup only | **Data:** Plugin startup (generated)
- Sent when EDMC detects game already running at launch
- Contains accumulated state from journal
- NOT in actual journal file

### `ShutDown`
**Frequency:** shutdown only | **Data:** Game shutdown (generated)
- Sent when game stops writing to journal without Shutdown event
- Indicates crash or unexpected exit
- NOT in actual journal file

## Event Data Sources

**Journal Events** - From `<game>/Saved Games/<commander>/Journal-YYYYMMDD-HHMMSS.log`
- Event-driven (fired when action occurs)
- Contains full data context
- May have delays (buffered)
- Accessed via `journal_entry()` callback

**Status Data** - From `<game>/Saved Games/<commander>/Status.json`
- Real-time (updated ~1/sec)
- Contains current state snapshot
- Minimal latency
- Accessed via `dashboard_entry()` callback

**State Dictionary** - Accumulated from journal_entry() events
- Persistent across events
- Updated with cumulative game state
- Passed to all callbacks
- Includes data from Cargo.json, Backpack.json, etc.
